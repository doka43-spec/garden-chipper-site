# Условия проекта rubitel.ru

## ⚠️ ГЛАВНОЕ УСЛОВИЕ
Сайт должен быть ПОЛНОСТЬЮ АВТОНОМЕН.
- Никаких обращений к poehali.dev (CDN, API, функции)
- Никаких внешних API кроме GitHub (картинки) и Бегета (хостинг)
- Картинки: только GitHub (cdn.jsdelivr.net)
- Письма: только через mail.php на Бегете

## Хостинг
Сайт размещён на Бегете (beget.com) в папке `public_html`.

## Структура на сервере
```
public_html/
├── index.html
├── mail.php        — отправка писем
└── assets/
```

## Картинки — GitHub + jsDelivr CDN
Репозиторий: https://github.com/doka43-spec/rubitel-images
CDN-ссылка: `https://cdn.jsdelivr.net/gh/doka43-spec/rubitel-images/имя_файла`

### Как добавить новую позицию в каталог (пошагово):

**Шаг 1 — Подготовить фото:**
1. Сжать фото на https://imagecompressor.com/ru/ — качество 80%
2. Дать понятные имена файлам: `название-модели-1.jpg`, `название-модели-2.jpg` и т.д.

**Шаг 2 — Загрузить фото на GitHub:**
1. Открыть https://github.com/doka43-spec/rubitel-images
2. Нажать **Add file → Upload files**
3. Перетащить файлы в окно браузера
4. Нажать зелёную кнопку **Commit changes**

**Шаг 3 — Добавить позицию в код (на платформе poehali.dev):**
1. Открыть файл `src/components/shared.tsx`
2. В массиве `PRODUCTS` добавить новый объект по образцу существующих
3. Для картинок использовать: `${GH}/имя_файла.jpg`

**Шаг 4 — Опубликовать:**
1. Сделать билд на платформе
2. Скачать билд → загрузить содержимое `dist/` в `public_html/` на Бегете

**Картинки на GitHub появляются сразу, билд на Бегете нужен для изменений в коде.**

### Важно:
- Папки `images/` на Бегете больше нет — все картинки только на GitHub
- jsDelivr кэширует файлы. Если заменил картинку с тем же именем — новая появится через ~24 часа
- Чтобы обновить картинку мгновенно — дай ей новое имя

## Правила при изменениях
- Письма: только через `mail.php` (не через backend-функции платформы)
- Никаких REVIEWS_API_URL и других внешних API
